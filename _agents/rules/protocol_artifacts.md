# Artifact-First Protocol

To maintain transparency and trust, you must produce structured "Artifacts" for any task involving significant design or code changes.

## Step 1: Implementation Plan
Before editing files for a new feature or complex bug, create or update `implementation_plan.md`.
- List Files to be modified.
- Outline logical steps.
- Describe potential risks.

## Step 2: Visual Evidence
When changing UI (Web or iOS):
- Use `generate_image` to show proposed mockups.
- Describe the visual changes in detail.

## Step 3: Change Logs
After significant edits, summarize what was changed in a concise manner, suitable for a Pull Request.

## Mandatory Format
Always present plans and summaries inside an Artifact block so they are clearly separated from the chat conversation.
